# Test Instructions

You are writing unit tests for the DentFlow API. Follow these conventions exactly.

## Stack

- **xUnit** — test framework
- **FluentAssertions** — assertions (`.Should().Be(...)`, `.Should().BeTrue()`)
- **NSubstitute** — mocking (`Substitute.For<T>()`, `Arg.Any<T>()`, `.Returns(...)`, `.Received(n)`, `.DidNotReceive()`)

## File & Naming Conventions

- Test project: `DentFlow.<Module>.Tests`
- Test file naming: `<Entity>CommandHandlerTests.cs` or `<Entity>QueryHandlerTests.cs`
- Test class naming: `<Entity><HandlerName>Tests` (e.g., `CreatePatientCommandHandlerTests`)
- Test method naming: `Handle_<Scenario>_<ExpectedOutcome>` (e.g., `Handle_DuplicateEmail_ReturnsConflictError`)

## Test Structure

```csharp
public class CreatePatientCommandHandlerTests
{
    // 1. Declare mock(s) at the top of the class
    private readonly IPatientRepository _repo = Substitute.For<IPatientRepository>();
    private readonly CreatePatientCommandHandler _sut;

    // 2. Initialize SUT in the constructor
    public CreatePatientCommandHandlerTests()
    {
        _sut = new CreatePatientCommandHandler(_repo);
    }

    // 3. Use a builder helper to create valid commands/requests
    private static CreatePatientCommand BuildCommand(string? email = null) => new(
        FirstName: "Jane",
        LastName: "Doe",
        Email: email,
        // ... all required params with sensible defaults
    );

    [Fact]
    public async Task Handle_NewPatient_CreatesAndReturnsResponse()
    {
        // Arrange
        _repo.GetByEmailAsync(Arg.Any<string>(), Arg.Any<CancellationToken>())
             .Returns((Patient?)null);
        _repo.GeneratePatientNumberAsync(Arg.Any<CancellationToken>())
             .Returns("P-000001");

        // Act
        var result = await _sut.Handle(BuildCommand("jane@test.com"), CancellationToken.None);

        // Assert
        result.IsError.Should().BeFalse();
        result.Value.FirstName.Should().Be("Jane");
        await _repo.Received(1).AddAsync(Arg.Any<Patient>(), Arg.Any<CancellationToken>());
    }
}
```

## What to Always Test

For every **command handler**, test:
1. Happy path — entity is created/updated/deleted and response is correct.
2. Duplicate/conflict path — returns the correct `ErrorOr` conflict error.
3. Not-found path (Update/Delete) — returns the correct `ErrorOr` not-found error.
4. Side-effect verification — e.g., `AddAsync` was called exactly once.

For every **query handler**, test:
1. Found path — returns the correct `ErrorOr<Response>`.
2. Not-found path — returns the correct `ErrorOr` not-found error.

## ErrorOr Assertions

```csharp
// Check success
result.IsError.Should().BeFalse();
result.Value.SomeField.Should().Be("expected");

// Check error type and code
result.IsError.Should().BeTrue();
result.FirstError.Code.Should().Be("Patient.NotFound");      // matches EntityErrors constant
result.FirstError.Type.Should().Be(ErrorType.NotFound);      // optional
```

## NSubstitute Patterns

```csharp
// Setup return value
_repo.GetByIdAsync(id, Arg.Any<CancellationToken>()).Returns(someEntity);

// Verify method was called
await _repo.Received(1).AddAsync(Arg.Any<Patient>(), Arg.Any<CancellationToken>());

// Verify method was NOT called
await _repo.DidNotReceive().AddAsync(Arg.Any<Patient>(), Arg.Any<CancellationToken>());
```

## What NOT to Do

- Do not use `Mock<T>` (Moq) — use `Substitute.For<T>()` (NSubstitute).
- Do not use `Assert.*` — use FluentAssertions `.Should()` methods.
- Do not test infrastructure (repositories, DbContext, EF) in unit tests.
- Do not test endpoints in unit tests — focus on command/query handlers.
- Do not share mutable state between tests.
