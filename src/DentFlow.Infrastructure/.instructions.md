# Infrastructure Layer Instructions

You are working inside `DentFlow.Infrastructure` — responsible for EF Core, repositories, and external service integrations.

## DbContext: `ApplicationDbContext`

Location: `Persistence/ApplicationDbContext.cs`

```csharp
public class ApplicationDbContext(
    DbContextOptions options,
    IMultiTenantContextAccessor multiTenantContextAccessor)
    : IdentityDbContext<ApplicationUser, ApplicationRole, Guid>
{
    // Add a DbSet<T> for every new entity
    public DbSet<Patient> Patients => Set<Patient>();

    protected override void OnModelCreating(ModelBuilder builder)
    {
        base.OnModelCreating(builder);
        // All entity configurations are applied from this assembly
        builder.ApplyConfigurationsFromAssembly(typeof(ApplicationDbContext).Assembly);
        // Global query filters (tenant isolation + soft delete) are applied here
    }
}
```

**When adding a new entity:**
1. Add `public DbSet<TEntity> <Entities> => Set<TEntity>();` to `ApplicationDbContext`.
2. Create an `IEntityTypeConfiguration<TEntity>` in `Persistence/Configurations/`.
3. Run `dotnet ef migrations add <Name> --project src/DentFlow.Infrastructure`.

## EF Entity Configuration Pattern

```csharp
// Persistence/Configurations/<Entity>Configuration.cs
public class PatientConfiguration : IEntityTypeConfiguration<Patient>
{
    public void Configure(EntityTypeBuilder<Patient> builder)
    {
        builder.ToTable("patients");   // snake_case table names

        // Required properties
        builder.Property(p => p.FirstName).HasMaxLength(100).IsRequired();

        // Enums stored as strings
        builder.Property(p => p.Status).HasConversion<string>().HasMaxLength(30);
        builder.Property(p => p.Gender).HasConversion<string>().HasMaxLength(20);

        // Indexes — always include TenantId as the leading column
        builder.HasIndex(p => new { p.TenantId, p.LastName, p.FirstName });

        // Unique indexes — add partial filter for soft-delete where appropriate
        builder.HasIndex(p => new { p.TenantId, p.Email })
               .HasFilter("\"Email\" IS NOT NULL AND \"IsDeleted\" = false");

        // Relationships
        builder.HasMany(p => p.EmergencyContacts)
               .WithOne(e => e.Patient)
               .HasForeignKey(e => e.PatientId)
               .OnDelete(DeleteBehavior.Cascade);
    }
}
```

**Rules:**
- Table names are lowercase snake_case.
- Every index must include `TenantId` as the leading column (tenant-scoped queries).
- Enums: always `HasConversion<string>()`.
- Soft-deleted records: use partial indexes with `IS NOT NULL AND "IsDeleted" = false`.

## Repository Pattern

**Location**: `Persistence/Repositories/<Entity>Repository.cs`

```csharp
// Primary constructor injection
public class PatientRepository(ApplicationDbContext dbContext) : IPatientRepository
{
    public async Task<Patient?> GetByIdAsync(Guid id, CancellationToken ct = default)
        => await dbContext.Patients.FirstOrDefaultAsync(p => p.Id == id, ct);

    public async Task AddAsync(Patient patient, CancellationToken ct = default)
    {
        await dbContext.Patients.AddAsync(patient, ct);
        await dbContext.SaveChangesAsync(ct);
    }

    public async Task SoftDeleteAsync(Patient patient, CancellationToken ct = default)
    {
        patient.SoftDelete();
        await dbContext.SaveChangesAsync(ct);
    }
}
```

**Rules:**
- Use primary constructor injection.
- Never inject `IServiceProvider` or create a scope manually.
- Never manually filter by `TenantId` or `IsDeleted` — EF global query filters do this automatically.
- Call `SaveChangesAsync` after every write operation (no Unit of Work abstraction needed).
- Lists/searches should use `AsNoTracking()` where possible for read queries.

## Registering Repositories

Add to `DependencyInjection.cs`:

```csharp
services.AddScoped<I<Entity>Repository, <Entity>Repository>();
```

## Identity Tables

ASP.NET Core Identity tables are renamed to snake_case (e.g., `asp_net_users`, `asp_net_roles`).  
`ApplicationUser` extends `IdentityUser<Guid>` and adds `TenantId`, `FirstName`, `LastName`, `IsActive`, `LastLoginAt`.  
`ApplicationRole` extends `IdentityRole<Guid>`.

## What NOT to Do

- Do not add business logic to repositories — only data access.
- Do not manually add `TenantId` filters in queries.
- Do not manually add `IsDeleted` filters in queries.
- Do not reference any module project from Infrastructure — only Domain.
