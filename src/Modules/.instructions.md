# Module Development Instructions

You are working inside a **DentFlow module**. Each module is a self-contained vertical slice of the system.

## Canonical Module Layout

```
DentFlow.<Module>/
  Domain/
    <Entity>.cs
    <Entity>Errors.cs
    <Entity>Enums.cs           # (if needed)
  Application/
    Commands/
      Create<Entity>Command.cs
      Create<Entity>CommandHandler.cs
      Create<Entity>CommandValidator.cs
      Update<Entity>Command.cs
      Update<Entity>CommandHandler.cs
      Update<Entity>CommandValidator.cs
      Delete<Entity>Command.cs
      Delete<Entity>CommandHandler.cs
    Queries/
      Get<Entity>ByIdQuery.cs
      Get<Entity>ByIdQueryHandler.cs
      List<Entity>sQuery.cs
      List<Entity>sQueryHandler.cs
    Interfaces/
      I<Entity>Repository.cs
    <Entity>Response.cs
    PagedResult.cs             # (only if module uses pagination)
  Endpoints/
    <Entity>CreateEndpoint.cs
    <Entity>GetByIdEndpoint.cs
    <Entity>ListEndpoint.cs
    <Entity>UpdateEndpoint.cs
    <Entity>DeleteEndpoint.cs
  DependencyInjection.cs
  DentFlow.<Module>.csproj
```

## Rules Specific to Modules

### Isolation
- Modules **never reference each other**. If shared data is needed, go through `DentFlow.Domain` or `DentFlow.Application`.
- Module `.csproj` references: `DentFlow.Domain`, `DentFlow.Application` — never another module.

### Domain Layer
- Entities inherit `TenantAuditableEntity` from `DentFlow.Domain.Common`.
- All properties: `private set`.
- Creation via `static Entity.Create(...)` factory method only.
- Mutations call `SetUpdated()` at the end.
- Error constants in `<Entity>Errors.cs` static class using `Error.NotFound / Error.Conflict / Error.Validation`.
- Enums are in `<Entity>Enums.cs` and stored as `string` in the DB.

### Application Layer
- Commands and queries are `record` types implementing `IRequest<ErrorOr<T>>`.
- Handlers use **primary constructor injection**.
- Validators extend `AbstractValidator<TCommand>` — they run automatically via the `ValidationBehavior` pipeline.
- `<Entity>Response` is a `record` with a `static FromEntity(Entity e)` factory. Never use AutoMapper.
- Repository interface lives in `Application/Interfaces/I<Entity>Repository.cs`.

### Endpoints Layer
- Inherit `Endpoint<TRequest, TResponse>` (or `EndpointWithoutRequest<TResponse>`).
- Inject `ISender` (not `IMediator`).
- `Configure()` must call `Roles(...)` with constants from `DentFlow.Domain.Identity.Roles`.
- Map `req` → command manually. Never pass the request DTO directly to MediatR.
- Use `SendCreatedAtAsync` (POST), `SendOkAsync` (GET/PUT), `SendNoContentAsync` (DELETE).
- Use `SendErrorsAsync` when `result.IsError`.

### Wiring New Modules
When creating a **new module**, these places must be updated:
1. Add the module assembly to `AddApplication(...)` in `Program.cs`.
2. Add the endpoint assembly to `o.Assemblies` in `AddFastEndpoints(...)` in `Program.cs`.
3. Register the repository in `DentFlow.Infrastructure/DependencyInjection.cs`.
4. Add the `DbSet<Entity>` to `ApplicationDbContext`.
5. Create the EF entity configuration in `DentFlow.Infrastructure/Persistence/Configurations/`.
6. Add a migration: `dotnet ef migrations add <MigrationName> --project src/DentFlow.Infrastructure`.
