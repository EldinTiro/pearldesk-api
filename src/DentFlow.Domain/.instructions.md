# Domain Layer Instructions

You are working inside `DentFlow.Domain` — the innermost layer. It has **zero external NuGet dependencies** and must stay that way.

## Base Classes

### `BaseEntity` (`DentFlow.Domain.Common`)
```csharp
public abstract class BaseEntity
{
    public Guid Id { get; private set; } = Guid.NewGuid();
}
```

### `TenantAuditableEntity` (`DentFlow.Domain.Common`)
```csharp
public abstract class TenantAuditableEntity : BaseEntity, ISoftDeletable
{
    public Guid TenantId { get; private set; }
    public DateTime CreatedAt { get; private set; } = DateTime.UtcNow;
    public DateTime? UpdatedAt { get; private set; }
    public bool IsDeleted { get; private set; }
    public DateTime? DeletedAt { get; private set; }

    public void SetTenant(Guid tenantId);
    public void SetUpdated();   // ← call at end of every mutation method
    public void SoftDelete();   // ← never hard-delete tenant data
}
```

**All tenant-scoped entities must inherit `TenantAuditableEntity`.**

## Entity Rules

1. All properties: `{ get; private set; }` — never public setters.
2. Private parameterless EF Core constructor: `private <Entity>() { }`.
3. Public creation only through `static Entity.Create(...)` factory.
4. All mutation methods call `SetUpdated()` at their end.
5. Never hard-delete; call `SoftDelete()` instead.
6. Collections are `ICollection<T>` with `private set` and initialized in the constructor or factory.

## Error Definitions

```csharp
// <Entity>Errors.cs
public static class PatientErrors
{
    public static readonly Error NotFound      = Error.NotFound("Patient.NotFound", "Patient was not found.");
    public static readonly Error AlreadyExists = Error.Conflict("Patient.AlreadyExists", "A patient with this email already exists.");
    public static readonly Error InvalidStatus = Error.Validation("Patient.InvalidStatus", "Invalid status transition.");
}
```

- Error codes follow the pattern `<Entity>.<PascalCaseName>`.
- Use `Error.NotFound`, `Error.Conflict`, `Error.Validation`, or `Error.Failure` from the `ErrorOr` package.
- Never define errors outside of the `<Entity>Errors` static class.

## Enums

- Defined in `<Entity>Enums.cs` within the module's `Domain/` folder.
- Stored as `string` in PostgreSQL (configured via `HasConversion<string>()` in EF configuration).
- Never store enums as integers.

## Roles (`DentFlow.Domain.Identity.Roles`)

```csharp
public static class Roles
{
    public const string SuperAdmin    = "SuperAdmin";
    public const string ClinicOwner   = "ClinicOwner";
    public const string ClinicAdmin   = "ClinicAdmin";
    public const string Dentist       = "Dentist";
    public const string Hygienist     = "Hygienist";
    public const string Receptionist  = "Receptionist";
    public const string BillingStaff  = "BillingStaff";
    public const string ReadOnly      = "ReadOnly";
}
```

Always reference role names from this class — never use string literals on endpoints.

## What NOT to Do

- Do not add NuGet packages to `DentFlow.Domain.csproj`.
- Do not reference `DentFlow.Application`, `DentFlow.Infrastructure`, or any module.
- Do not add business logic that depends on infrastructure concerns (DB, HTTP, etc.).
